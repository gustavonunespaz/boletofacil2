Boleto Fácil 2

Reescrita em Java com arquitetura limpa para automatizar boletos, lotes de envio e geração de artefatos.

Objetivos do MVP

Processar boletos
  • Extrair dados do PDF (cliente, endereço, parcela, vencimento).
  • Gerar PDF final organizado por ano/mês.
Gerar lote Correios
  • PDF compilado com boletos selecionados.
  • Exportar TXT e Excel com dados de clientes.
Ajustar endereços
  • Atualizar endereço normalizado do boleto.
  • Regerar PDF processado após a correção.

Arquitetura (Clean/Hexagonal)

  • Camada de domínio independente de frameworks, aplicando SOLID e valor-objects.
  • Casos de uso orquestram fluxos e dependem de ports (interfaces).
  • Adapters de infraestrutura (PDFBox, POI, JPA) implementam as ports.
  • Interfaces REST ou CLI acessam os casos de uso.

Pilha tecnológica

Linguagem
  • Java 17/21, Maven, testes com JUnit 5 e Mockito.
Infra
  • Spring Boot (Web, Data JPA), PostgreSQL e H2 para dev.
Arquivos
  • Apache PDFBox para PDF e Apache POI para Excel.

Documentação completa em ARCHITECTURE.md. Princípios ACID e Código Limpo guiando a evolução.
